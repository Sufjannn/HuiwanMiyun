# 密云旅游应用更新说明

## 项目概述

本项目是一个密云旅游景点的前端应用，包含集章打卡、景区地图、接驳车预约、论坛等功能。

## 主要更新内容

### 1. 登录系统
- 实现了手机号+验证码登录功能
- 登录状态持久化存储
- 未登录用户无法访问需要权限的功能

### 2. 集章系统
- 支持扫描二维码跳转到对应主题页面
- 徽章解锁状态持久化存储
- 未登录用户显示0/4的集章进度
- 首页和集章页面同步显示徽章状态

### 3. 导航菜单
- 右上角添加了用户菜单
- 包含"我的预约"、"个人信息"、"设置"、"退出登录"选项
- 登录后显示用户信息

### 4. 预约系统
- 接驳车预约需要登录才能使用
- 支持查看、修改、取消预约
- 预约信息持久化存储

### 5. 个人信息管理
- 支持查看和修改个人资料
- 包括姓名、手机号、身份证号、邮箱

### 6. 权限控制
- 预约、集章、论坛页面需要登录才能访问
- 扫码打卡功能需要登录才能使用

## 技术实现

### 前端技术栈
- HTML5 + CSS3 + JavaScript
- 响应式设计
- 本地存储 (localStorage, sessionStorage)
- QR码扫描 (jsQR库)

### 数据持久化
- 用户信息存储在 localStorage 中
- 集章进度存储在 localStorage 中
- 预约信息存储在 localStorage 中
- 扫码状态使用 sessionStorage 传递

### 关键功能
1. **扫码解锁**：使用 jsQR 库扫描二维码，识别主题 URL，跳转对应页面
2. **登录验证**：手机号+验证码登录，支持验证码倒计时
3. **状态同步**：页面可见性变化时自动更新徽章状态
4. **权限控制**：导航和功能访问前检查登录状态

## 测试方法

1. **登录测试**：点击右上角菜单，使用手机号登录
2. **集章测试**：点击集章页面的"扫码打卡"按钮，扫描对应二维码
3. **预约测试**：登录后预约接驳车，查看预约记录
4. **权限测试**：未登录状态下尝试访问需要权限的页面

## 注意事项

- 扫码功能需要浏览器相机权限
- 验证码为模拟发送，可输入任意6位数字
- 数据存储在本地，清除浏览器数据会重置所有状态
